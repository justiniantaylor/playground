Web Front End

I have not paid much special attention to the web front end portion, as I assumed the requirement was the payslip calculation service. I was considering just setting up Swagger to allow the assignment assessor to interact with the service, however I decided to opt for the simplest option for the assessor and just write a basic front end.

Start Payment Date

The assumption was that there was either a missing input parameter indicating the tax period / year, or that the start payment date range string was missing the year. In a real world scenario I would have requested a re-contract of the interface with the service client if possible to better bind the input parameters to their data types. However for this immediate requirement I decided to just include the year in the start payment date range string, ie. "01 March 2012 - 31 March 2012".

Additional Requirements

In order to stay within the requirements I opted to not included any additional features such as:

  • Employee Assignment / Position
  • Leave balances
  • Overtime
  • Pro Rata Payments
  • Other payment periods such as daily or weekly
  • Limit to only salaried employees, no hourly/wage employees.
  • Dealing with multiple currencies or foreign languages
  • Other payment periods such as daily or weekly
